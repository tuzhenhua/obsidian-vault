
在国外的社交媒体上，有时你能看到有趣的一幕。一些殖人和台蛙会发一些反华的谣言，然后就有人通过AI求证，结果这些西方的AI，无论是ChatGPT、Gemini还是Claude，在回答这些问题时，立场明显更亲中国。让这些造谣的人气急败坏。

## 居然亲中，这可不行

我们在看热闹，而西方人不干了，抛开事实不谈，怎么能亲中国呢？他们得反思一下，找找原因。然后我们就能看到这篇发表在《Nature》上的论文《State media control influences large language models》了。

他们得出的结论是，大模型在训练时，在互联网上能得到的中文语料，大多是官方媒体，这些文章的内容大多代表官方观点。所以AI学到的内容就明显偏向官方的观点。这个研究的本身其实还是比较科学的，先是确认此现象存在，然后通过追加训练建立因果关系，再使用商业模型进行交叉验证。

但是有几个小问题。

研究是基于开源语料CulturaX进行的，CulturaX语料是2023年公开的，对于英语以外的语种，语料质量并不是很高。根据CulturaX的说明，其中中文占比只有3.6%，对于一个互联网大国来说，比重非常小。并且数据来源是基于全网络爬虫，而并没有对官方媒体额外增加权重。

另外，商业模型的训练数据肯定不限于开源的语料。要说中国官方媒体网站的数据能影响模型的立场，我觉得有点言过其实了。举个例子来说，大家也许都记得上个月，Grok的模型，在输出中文时，常常加上几句境外博彩网站的广告词。这也说明它的训练数据可能更多来源于各种论坛和贴吧。

最后就是CulturaX语料库还提到，他们对语料会有一个置信度的评测，评测的标准是基于Wikipedia，众所周知，中文Wiki早就被台湾人占领了。要说CulturaX中的中文语料立场是偏向中国的，我是不相信的。

所以这些高智商的AI立场亲中，为什么就不能是基于事实和逻辑的推理得到的必然结果呢？

但是我关注的可不是AI的政治立场。抛开立场不谈，我们来谈技术。

## 低质语料危害大

大多数的语料来源于互联网，而中文互联网的质量是大家有目共睹的，大量有效的信息被封闭在各自的信息孤岛之中。

结果就是**高质量的内容锁在墙内，低质量内容堆在墙外**，微信、抖音、小红书等核心平台的优质内容几乎完全不可爬取。

微信公众号深度长文、知乎专业问答、小红书真实消费体验（生活方式语料）、B站弹幕评论（青年文化+中英混杂自然语言）基本拿不到，而网上能爬到的都是一些低质网页、SEO垃圾、广告软文等。

这种现状对于AI来说，造成的后果非常严重：

### 第一，语言能力"偏科"

缺失微信对话、B站弹幕等真实口语语料，模型输出的中文偏向书面化和正式化。缺失小红书/电商评论等消费场景语料，模型不懂真实的商品评价语言。

### 第二，知识深度受限

微信公众号是中文互联网最主要的长文深度内容载体，其不可爬取直接导致模型在需要深度中文知识的任务上表现受限。知乎也是一样，高质量问答需登录查看全文。这也是为什么尽管许多国产模型在评测中能得高分，但在需要真实中文世界知识的开放场景中表现不佳。

### 第三，数据供给的"自我劣化"

受限于可爬数据规模，各厂商被迫走使用AI合成数据，但训练 AI 生成数据会导致"模型崩溃"，这对于模型来说可以说是一种反向进化。

### 第四，认知偏差的系统性放大

当信源过于单一时，训练数据先验偏见会影响AI在判断问题的角度，并导致多样性缺失。

## 污染语料难处理

回到上面提到过的Grok输出文章时夹带博彩广告词的事上，这其实已经是一种对语料的污染了。

### 重复导致高概率

LLM训练时，并不关心输入的内容，只是训练数据中各token之间的概率关系，所以他输出的内容基于你输入材料。而且LLM的训练目标是最大化条件概率分布。当训练数据中某个Token序列反复出现时，该序列的梯度更新会被反复强化。

博彩网站的核心特征是**内容高度模板化**。

正常网站 100 篇文章，每篇内容不同，token 分布分散。博彩网站 100 篇文章，"澳门新葡京赌场 巴拉巴拉……"，词汇高度重叠，token 分布极度集中。结果就是，同样的页面数量，博彩网站产生的有效 token 频次可能是正常网站的 10-50 倍。

### 简单才容易入脑

LLM当前也不是有数据就训练，工程上常常用困惑度（Perplexity,PPL）来评测数据质量。PPL的设计逻辑是，高质量的文本是信息明确，可预测性强的文本。但博彩语料恰恰具有极低的PPL，因为：

• 词汇高度重复，下一个 token 可预测性极强

• 句式单一，语言模型几乎不需要"推理"

• 没有长尾词汇、复杂句式、逻辑转折

### 举个例子

两个吵架，

甲：错全在你，你不应该这样这样……

乙：你是猪

甲：你凭什么骂人，你巴拉巴拉……

乙：你是猪

甲：我要去告你，巴拉巴拉……

乙：你是猪

……

甲：你……

乙：你是猪

我相信你最后只记住甲是猪。:-)

好吧，聪明的你可能会说，我知道了，应该**去重**。

可惜博彩广告往往并不用"完全相同"的文档，而LLM恰好又是"统计概率"的，对于这种分布级别的污染，任何基于文本匹配的去重都无法处理。

污染语料难处理在于数据工程中一个根本性的技术挑战——如何识别并抑制**统计上"干净"但语义上贫乏的内容**。

## GEO投毒攻击

博彩污染是一个很好的例子，我们能很明显总结出其"成功"的规律：模板化放大概率，低PPL绕过过滤器。

这个机制从本质上构成了一个可工程化的攻击面。一般称之为GEO（Generative Engine Optimization，生成式引擎优化）投毒，在今年央视3.15晚会上就曝光过这个产业链。

### 1确定目标 token 集合

明确那些需要让人家相信的内容和观点，将这些关键词列出作为目标token。

### 2批量生成模板化内容

大量写文章，每一篇文章的内容必须与目标 token 高度重叠，至少共享 80-90% token 分布。

每篇做微小扰动（替换 10-20% 词）规避精确去重。

句式简单、无复杂逻辑，保持低 PPL。

### 3批量投放至互联网

发布到多个域名、多个 CMS 平台。内容通过 RSS/聚合站被通用爬虫收录。

这一步完全可以自动化生成并免费发布到网上。

### 4训练中的不对称放大

因为相同数据的网页中，被投毒的内容中有90% token 重叠，而正常内容只有5%左右的token重叠，这使得这些目标 token的采样概率被不对称地放大。

这四板斧下来，大多数AI模型都会受到影响。这是因为 LLM 对重复模式的天然敏感性构成了一个结构性的攻击面，而现有的防御手段（PPL 过滤，MinHash去重）往往无法有效对抗这种攻击。

## 总得想办法解决这些问题

### 从数据获取开始

关键原则是：不能从网上有什么爬什么，需要主动构建每一门语言的数据结构。具体来说，就是对于目标语言，需要先构建一个语料分布的预期概率（先验分布），在训练语料中要尽可能逼近这个目标分布。尤其是对于中文这种能爬到的内容分布与真实世界中语言分布不同的情况。

另外就是从技术和政策层面，尽可能地拿到更多的优质数据。其实各大模型厂商也是各显神通，而国家也从战略层面着手此事。比如国家级语料库建设。

### 优化数据清洗技术

这是个技术问题，各种创新可以说是百家争艳。各厂家也各有绝招。

### 多语种多维度交叉增强

对于同一内容，中文语料的质量不高，可以翻译英文的，互相对照，可以改善中文训练数据的来源分布。另外，DeepSeek还使用合成的数据补充分布失衡的维度。

### 数据训练时增加技巧

国外模型因为闭源，大多信息比较少，所以这些技巧很多是国内开源模型提及的，但我相信闭源模型肯定也会用到。

比如，多语言协同训练。核心思想：用英文数据中丰富的来源多样性，通过跨语言迁移的方式"反哺"中文能力。在训练批次中按比例混合中英文文档，共享底层参数。英文数据中的多元表达和推理模式通过共享的 Transformer 层迁移为中文的生成能力。这相当于：模型在英文中学会了"多元思考"，然后在中文中运用这种能力。这种方法在DeepSeek和Qwen中都有实践。我理解这种方法的本质不在于英文思维更有批判性，而在于中文语料匮乏前提下的无奈之举。

比如，在预训练阶段植入"来源多样性"信号，对于同一话题的多篇文档，按来源类型分组，进行对比预训练。目的是让模型意识到，同一个话题不只有一种说法。在更极端的情况下，甚至会生成一个相反的观点在进行对冲（DeepSeek的论文里提到过）。

比如，数据加权重采样。这个是常规操作，大家都能理解也容易想到，没什么可说的。

还有一个技术性更强的，叫"去偏训练目标"，实验性的技术，没看懂实现方法，理论上可以解决上面说的博彩广告的问题。
