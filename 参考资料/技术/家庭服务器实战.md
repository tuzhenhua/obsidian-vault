# 家庭服务器实战：一台主机干三件事

> **场景**：桌子下面有台配置不错的主机，家里还有笔记本电脑和电视机。让这台主机同时承担 **游戏串流引擎 + NAS 文件中心 + 家庭影音核心**，笔记本和电视通过网络调用主机的算力，手机自动备份照片到主机。

---

### 🎮 旧网吧 vs 新家庭网吧

二十年前泡网吧，一排排长得一模一样的机器，网线连着交换机——**共享的是网线，各自算各自的**。每台机子都有 CPU、显卡、硬盘，但人一走，上面的东西全还原。

今天搭的这套，本质是**家庭网吧**的终极形态：

| | 旧网吧 | 新家庭网吧 |
|---|--------|-----------|
| 算力 | 每台机器自己扛 | 主机一台全包，**共享显卡和 CPU** |
| 存储 | 重启就还原，存个东西靠 U 盘 | NAS 统一存，**所有设备共享同一块硬盘** |
| 游戏 | 每台都得装一遍 | 主机装一份，**笔记本和电视只管显示画面** |
| 设备 | 全是一样的塔式机箱 | 笔记本、电视、手机、平板，**什么能亮屏什么就能接进来** |
| 位置 | 人去找机器，下机就没了 | **机器等你回家**，走到哪跟到哪 |
| 上网 | 百兆共享，打个 CS 都跳 ping | 千兆内网 + 5GHz WiFi，串流延迟不到 10ms |
| 隐私 | 旁边坐一陌生人盯着你屏幕 | 自己家、自己用、**数据全在自己硬盘上** |

以前一台万元主机是一个人用一次。现在一台万元主机是全家用、随时用、在外面也能用。**从位置绑定变成了身份绑定**——你在哪，算力就流到哪。

## 一、游戏串流：主机渲染，笔记本/电视显示

### 软件选择

| 方案 | 延迟 | GPU兼容性 | 易用性 | 免费 | 推荐度 |
|------|------|-----------|--------|------|--------|
| **Apollo + Moonlight** | ⭐⭐⭐ 最低 | NVIDIA/AMD/Intel | 中等 | ✅ | ⭐⭐⭐⭐⭐ |
| Steam Remote Play | ⭐⭐ | 全支持 | 极简 | ✅ | ⭐⭐⭐⭐ |
| Parsec | ⭐⭐ | 全支持 | 极简 | ⚠️ | ⭐⭐⭐ |

**推荐 Apollo + Moonlight**：开源、延迟最低、自带虚拟显示器，主机不需要插物理显示器。

### 主机端：安装 Apollo

Apollo 是 Sunshine 的增强分支，原生支持虚拟显示器。

1. GitHub 搜索 `ClassicOldSong/Apollo`，下载最新 `.exe`
2. 安装后任务栏出现图标，打开 Web 管理界面（`https://localhost:47990`）
3. 设置用户名/密码，记下 PIN 码

**核心配置：**

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 视频编码 | NVIDIA→NVENC / AMD→AMF / Intel→QSV | 硬件编码零CPU开销 |
| 分辨率 | 等于客户端屏幕原始分辨率 | 如 1920×1080 或 2560×1600 |
| 帧率 | 60 FPS（或120如屏幕支持） | 匹配客户端屏幕 |
| 码率 | 有线 80-150 Mbps / WiFi 30-60 Mbps | 越高画质越好 |
| 编码预设 | P1（最快） | 优先低延迟 |
| 虚拟显示器 | ✅ 启用 | 关掉物理显示器也能串流 |

### 笔记本端：安装 Moonlight

1. [moonlight-stream.org](https://moonlight-stream.org) 下载客户端
2. 打开自动扫描局域网内主机
3. 输入主机 PIN 码配对
4. 配对成功即可看到主机桌面

**Moonlight 端优化：**
- 视频解码选 **Hardware Accelerated**（硬件解码）
- 笔记本有独显（哪怕入门级）开启 **HEVC/H.265**，同样画质省 30-40% 带宽
- 关闭 V-Sync（Moonlight 自己有帧同步）
- 关闭笔记本蓝牙、VPN（减少WiFi干扰）

### 纯 WiFi 场景优化（笔记本没有网线口）

- **必须用 5GHz**，不要连 2.4GHz（干扰太大）
- 路由器后台单独开 5GHz SSID（如 `MyWiFi_5G`）
- 笔记本和路由器之间尽量少隔墙
- 用 WiFi Analyzer 扫信道，固定到空闲信道
- 关闭后台下载、Windows Update、OneDrive 同步

### 远程开机：Wake-on-LAN

**主板 BIOS 设置：**
- Wake on LAN → Enabled
- PCI-E Wake → Enabled
- 关闭 ErP Ready（会切断网卡供电）

**Windows 设置：**
- 设备管理器 → 有线网卡 → 属性 → 高级：
  - Wake on Magic Packet → Enabled
  - 关机网络唤醒 → Enabled
- 电源选项 → 取消"启用快速启动"

**唤醒工具：**
- 手机 App 搜 "Wake on Lan"，输入主机 MAC 地址和内网 IP
- Moonlight 客户端在检测到主机离线时自动发送 WoL 数据包
- 路由器后台（华硕/小米等）自带 WoL 功能

> 记下主机有线网卡 MAC 地址：`ipconfig /all` → 物理地址（12位十六进制）

## 二、NAS 文件共享：让所有设备都能读写主机硬盘

### SMB 共享设置

**主机端：**

```powershell
# 创建共享文件夹结构
New-Item -ItemType Directory -Path "D:\NAS" -Force
New-Item -ItemType Directory -Path "D:\NAS\照片备份" -Force
New-Item -ItemType Directory -Path "D:\NAS\电影音乐" -Force
New-Item -ItemType Directory -Path "D:\NAS\文档资料" -Force
```

1. 控制面板 → 用户账户 → 新建本地账户 `nasuser`，设密码
2. 右键 `D:\NAS` → 属性 → 共享 → 授权给 `nasuser`（读写）
3. 控制面板 → 启用或关闭 Windows 功能 → 勾选 SMB 1.0/CIFS

**路由器上固定主机 IP：**
- 进路由器后台 → DHCP 设置 → 绑定主机 MAC 到固定 IP，如 `192.168.1.100`

**笔记本访问：**
- 文件管理器地址栏输入 `\\192.168.1.100\NAS`
- 输入 `nasuser` 账号密码
- 右键 → 映射网络驱动器，之后就像本地硬盘一样用

### 手机照片自动备份

#### 方案 A：FolderSync（Android 首选）

| 设置项 | 值 |
|--------|-----|
| 账户类型 | SMB/CIFS |
| 服务器地址 | `192.168.1.100` |
| 共享文件夹 | `NAS/照片备份` |
| 用户名/密码 | `nasuser` |

同步任务配置：
- 本地文件夹：`DCIM/Camera`
- 远程文件夹：`照片备份/手机型号/`
- 方向：仅上传
- 触发：每天定时 + 充电时 + 连上家里WiFi时
- 不删除本地文件

#### 方案 B：Syncthing（多设备双向同步）

1. 主机装 [Syncthing](https://syncthing.net) Windows 版
2. 手机装 Syncthing App
3. 两台设备配对，设置 `DCIM/Camera` ↔ `D:\NAS\照片备份` 双向同步
4. 局域网自动发现，回家连上 WiFi 就开始传

#### 方案 C：Immich（Google Photos 替代品）

1. 主机装 Docker Desktop for Windows
2. 跑 Immich 容器（GitHub: immich-app/immich）
3. 手机装 Immich App，自动备份
4. 浏览器打开 `192.168.1.100:2283` → 按人脸/地点/时间线浏览照片墙

### 电源计划调优

```
控制面板 → 电源选项 → 高级电源设置

硬盘     → 在此时间后关闭硬盘 → 20 分钟
睡眠     → 在此时间后睡眠     → 从不
```

**功耗参考：**

## 七、故障排查速查

| 问题 | 原因 | 解决 |
|------|------|------|
| Moonlight 扫不到主机 | 防火墙拦截 | 主机放行 Apollo 端口（47984-48010 TCP/UDP） |
| 画面卡顿/马赛克 | 码率太低或WiFi干扰 | 提高码率；切到5GHz；靠近路由器 |
| 延迟高(>30ms) | 网络跳数多 | 主机插网线；关笔记本蓝牙和VPN |
| 声音不同步 | 编码缓冲大 | Apollo 降低编码缓冲 |
| 手柄不识别 | USB透传未开 | Moonlight 设置→输入→勾选手柄驱动 |
| 笔记本访问不了NAS | IP变动或权限 | 检查主机IP；确认 `nasuser` 有读写权限 |
| WoL 唤不醒 | 快速启动阻止 | 关闭 Windows 快速启动 |

## 八、外网访问：出门在外也能用家里服务器

> 前面的方案全部跑在局域网内。加上 Tailscale 虚拟组网后，手机 / 笔记本不管在哪儿，都像回到了家里。

### Tailscale 一键组网（免费，3人100台设备以内够用）

**原理**：所有设备获得一个 `100.x.x.x` 的虚拟 IP，通过加密隧道直接 P2P 互联，不走公网暴露。

| 设备 | 安装方式 |
|------|---------|
| 主机（Windows） | [tailscale.com](https://tailscale.com) 下载 `.exe`，安装后登录（GitHub/Google 账号） |
| 笔记本电脑 | 同上，装 Tailscale 客户端并登录同一账号 |
| 手机（Android） | Google Play / APKPure 搜 "Tailscale" |
| 手机（iPhone） | App Store 搜 "Tailscale" |

装完后打开 Tailscale 面板（任务栏图标 / App），能看到所有设备的 **Tailscale IP**（形如 `100.xxx.xxx.xxx`），记下主机的那个 IP。

### 各功能在外网的使用方式

把原来所有写 `192.168.1.100` 的地方，换成主机的 **Tailscale IP**。

| 功能 | 内网地址 | 外网地址（Tailscale） |
|------|---------|----------------------|
| SMB 文件访问 | `\\192.168.1.100\NAS` | `\\100.x.x.x\NAS` |
| RDP 远程桌面 | `192.168.1.100` | `100.x.x.x`（RDP 直接填 IP） |
| 照片备份（FolderSync） | SMB 服务器填 `192.168.1.100` | 改成主机 Tailscale IP |
| Jellyfin 看视频 | `192.168.1.100:8096` | `100.x.x.x:8096` |

### 手机照片备份在外网

FolderSync 同步任务设置调整：

| 设置项 | 内网值 | 外网修改 |
|--------|--------|---------|
| SMB 服务器地址 | `192.168.1.100` | → 主机 Tailscale IP |
| 触发条件 | 每天 + 充电时 + 连上家里WiFi | → **改为：连上 WiFi 时** |

> ⚠️ 不要在外网用移动数据自动备份，会跑流量。改成"连上 WiFi 时"触发，到了酒店 / 咖啡店连上 WiFi 才开始传。

### 家庭影音服务器：Jellyfin（外网看视频不卡）

SMB 直接播高码率视频，受限于家宽上行带宽，4K 大概率卡。搭个 Jellyfin 服务器，它会**根据你当前网速实时转码**，网慢自动降画质，保证流畅。

#### 主机部署（Docker，3分钟）

```powershell
# 前提：已装 Docker Desktop（和 Immich 一样）
docker run -d `
  --name jellyfin `
  -p 8096:8096 `
  -v "D:\NAS\电影音乐:/media" `
  --restart unless-stopped `
  jellyfin/jellyfin
```

浏览器打开 `http://localhost:8096`，初始化向导：
- 添加媒体库 → 路径填 `/media`
- 内容类型选"电影"或"音乐"
- 元数据语言选中文
- 硬件加速：NVIDIA NVENC / Intel QSV / AMD AMF（有独显就开，转码省 CPU）

#### 外面怎么看

| 设备 | 怎么做 |
|------|--------|
| 笔记本 | 浏览器打开 `http://100.x.x.x:8096`（主机 Tailscale IP），和在家里刷 Netflix 一样 |
| 手机 | 装 Jellyfin App（iOS / Android 都有），服务器地址填 `http://100.x.x.x:8096` |
| 电视盒子 | 装 Jellyfin 客户端，同样填 Tailscale IP |

> Jellyfin 免费、开源、无广告，体验对标 Plex 但不收费。手机 App 也免费，不像 Plex 要买 Pass。

### 硬件加速转码（Jellyfin 设置）

主机有独显的话，让 Jellyfin 用显卡转码，CPU 零开销：

```
Jellyfin 后台 → 控制台 → 播放 → 硬件加速
  - 硬件加速：NVIDIA NVENC（N卡）/ Intel QuickSync（核显）/ VAAPI（A卡）
  - 启用硬件编码：✅ 全部勾上
  - 允许 HEVC 编码：✅
```

### 外网各功能体验总结

| 功能 | 外网能用吗 | 实际体验 |
|------|-----------|---------|
| 存取文档（SMB） | ✅ | 秒开，和在家里一样 |
| 照片备份 | ✅ | 连 WiFi 自动传，稳 |
| 看 1080p 视频（Jellyfin） | ✅ | 流畅不卡，自动匹配网速 |
| 看 4K 视频（Jellyfin） | ✅ | 自动转码 1080p，画质依然不错 |
| 看 4K 原片（SMB 直播） | ⚠️ | 家宽上行不够，会卡 |
| 远程桌面（RDP） | ✅ | 延迟低，改配置装软件完全够用 |
| ~~游戏串流~~ | — | 上行带宽不够，放弃 |

## 九、外网故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| Tailscale 连不上 | 网络环境阻断了 P2P | Tailscale 会自动走中继（DERP），慢但能用；试试关 VPN 或换网络 |
| SMB 在 Tailscale 下很慢 | SMB 协议对大延迟敏感 | 正常现象，小文件 OK，大文件耐心等 |
| Jellyfin 转码后还是卡 | 硬件加速没生效 | 检查 Docker 是否允许 GPU 访问；先试试软解能不能播 |
| 手机在外面 FolderSync 失败 | SMB 地址还是内网 IP | 确认填的是主机 Tailscale IP，不是 192.168.x.x |
| Jellyfin 手机 App 连不上 | Tailscale 未连接 | 打开手机 Tailscale App，确认已连接且有虚拟 IP |

---

> **一句话总结**：主机插着网线和电源线丢在桌子下面，不开显示器；笔记本和电视通过 Moonlight 调用主机的显卡跑游戏，手机照片自动备份到主机，全家设备通过 SMB 共享文件；出门在外 Tailscale 一连，NAS 照用、照片照传、Jellyfin 照看不误。一台机器，物尽其用。

## 🥚 彩蛋：一条穿越三十年的网线

> *以下内容摘自 2026年5月31日与 Claude 的聊天记录。*

**1996 年的网吧**，红警不是双击图标打开的，得手敲：

```
C:\> cd redalert
```

**1998 年**，网吧里打星际争霸，网卡得兵都控不住。有个小子盯着机箱后面盘根错节的网线想：能不能**用激光组网**？每台机器屁股后面装个激光发射器和接收器，光速传数据，连网线都省了。

那个年代大部分人连以太网都搞不清楚，网吧还在用同轴电缆或 10Mbps 双绞线。冒出这个念头确实超前得离谱。

但回头一看，这条思路没跑偏——它对应的真实技术叫 **自由空间光通信（FSO）**，两栋楼之间不拉光纤，架一对激光收发器就能点对点高速联网。Starlink 卫星之间通信用的就是激光。甚至今天的 **Li-Fi（光保真技术）**，本质上就是可见光无线通信。

**2026 年**，当年那个敲命令行打红警、幻想激光联网的小子，用 Apollo 在笔记本上 10ms 延迟串流 3A 大作，用 Tailscale 在外面看家里 Jellyfin 的电影，手机照片自动飞回主机 NAS。

从 `RA95.EXE` 到 Moonlight 串流，从激光幻想到家宽千兆。三十年，网线换了，想联网的心没换过。

> 那条想象中的激光——终于还是联通了。
