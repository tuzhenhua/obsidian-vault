# 善作者不必善成——用认知框架分析

> 善作者不必善成，善始者不必善终。这是乐毅给自己一生下的注脚——也是最狠的一刀关系主义操作。它把"做"和"成"拆开了，告诉你两件事之间没有必然联系。**能力是你自己的，结果是系统和你的关系。** 从来不是"你够不够好"，是"你和条件配不配"。

---

## 一、核心事实——乐毅的"做得成却做不完"

出处：《史记·乐毅列传》《战国策》。

故事背景：

乐毅是燕国大将，率五国联军攻齐，**连下七十余城**，齐国只剩即墨和莒两城未破。然后燕昭王死，燕惠王继位，齐国的田单用反间计，燕惠王派骑劫替换乐毅。乐毅怕被诛杀，逃往赵国。骑劫接管后，田单反攻，燕军大败，所有城池全部丢失。

七十余城都打下来了，最后两城打不下来——不是因为乐毅没能力（善作），是因为信任这个系统条件变了（不善成）。

乐毅写给燕惠王的信里说："善作者不必善成，善始者不必善终。"——他是在说：**我不是不能打完，是你不让我打完。** 这话很客气，但刀很锋利：他把"我的能力"和"你的配合"分开了，不背结果不成的锅。

---

## 二、本质主义 vs 关系主义——"作"和"成"不是同一个变量

### 字面上的关系主义

这句话的精髓就在一个"不必"——"不一定"。

本质主义的思维是：有能力 → 能做成。这是一个因果链：能力是因，成果是果。如果做不成，那就是能力不够。

关系主义的思维是：**有能力 × 条件匹配 = 做成。** 能力和成果之间不是直接因果，是被系统条件调制的。乐毅能打下七十城（能力验证了），但打不下最后两城（条件变了）。

本质主义问："你行不行？"
关系主义问："在什么条件下你行？"

### 善作 vs 善成

| | 善作 | 善成 |
|--|------|------|
| **对象** | 你和你做的事 | 你和系统的关系 |
| **控制范围** | 你一个人可控 | 你需要系统配合 |
| **校验标准** | 做得好不好（质量） | 做成了没（结果） |
| **时间** | 当下 | 持续到结果出现 |
| **典型例子** | 文章写得透彻（能力） | 文章到不了500（系统不配合） |

原文的框架连接说得好："善作是个人能力，善成是系统能力。"——换成框架语言：**善作是你在局部的关系中操作得好，善成是整体系统的关系链都通了。**

### "做"和"成"之间的那个缝隙

"善作"和"善成"之间有一个缝隙——**这个缝隙就是不确定性的来源，也是校验的困难所在。**

```
善作 ───→ [条件A × 条件B × 条件C × ……] ───→ 善成
```

你做得好（善作），但条件A变了（信任没了）、条件B没到位（别人不配合）、条件C不成立（时机不对）——结果就是不善成。

这不是你不行，是**所有条件同时成立的概率本来就低**。这和吉凶悔吝的"四分之三不是吉"是同一条逻辑——在复杂系统中，大多数行动不会得到预期结果。

## 三、语迷——"善"字把两件事装进了一个词

### 同一个"善"的陷阱

"善作"和"善成"共享一个"善"字，这在语言上给人一个错觉：这两件事是同类——都是"做得好的"。

但"善作"的"善"是能力层面的，"善成"的"善"是结果层面的。**两个"善"不是一个善。** 一个形容你的动作质量（做得漂亮），一个形容系统的输出状态（结果达成了）。

**语迷操作就在这里：同一个字让你以为它们是同一个性质的事，但实际是完全不同层面的判断。**

| 语迷步 | 操作 | 实例 |
|--------|------|------|
| **Lv.1 词义锁** | "善"字统一了两件事的外表 | 善作和善成共用同一个形容词 |
| **Lv.2 关系定位锁** | 把责任感定位到个人 | 你没做成 = 你不够善——从"善作≠善成"滑到"不善成=不善作" |
| **Lv.3 叙事封锁** | 只看结果不看过 | "结果不好，那前面的过程也有问题"——后见之明的标准化叙事 |

最典型的是后见之明叙事：乐毅没打下最后两城 → 后世解读"他是不是本来就不行？"——因为他"不善成"，所以他的"善作"也被怀疑了。这正是"善"这个字造成的语言陷阱——**你把结果标签贴回过程，过程的质量就被覆盖了。**

### 乐毅的解法：把两个"善"分开

乐毅在信里做的事情就是拆这个锁。他说"善作者不必善成"——他把两个"善"拆开了：**我作的部分是善的（七十城的战绩），但成不是由我决定的（最后两城的事不是你让我做完的）。**

他不争"我有没有善成"——他承认没成。他争的是：**你不能用"没成"来否定"善作"。** 这是语迷解构的标准操作——你说"A"，我告诉你"A"不是你想的那个意思。

## 四、盲人摸象——乐毅在哪一层

| 层次 | 状态 | 乐毅的位置 |
|------|------|-----------|
| **一盲·真盲** | 以为有能力就一定能成 | 天真者——"我这么强，怎么可能失败" |
| **二盲·摸局部当整体** | 知道能力不等于成功，但归因于单个因素（运气/敌人太强/队友太弱） | 归因者——"就差那么一点" |
| **三盲·看见自己盲** | 知道"作"和"成"之间是多因素系统交互，自己能控制的部分有限 | **乐毅在此**——他说"善作者不必善成"，承认了能力的局限性 |
| **四盲·"可能没有成"** | "成"本身也是临时的——七十城也不是永远的，最后全丢了 | 乐毅没到——他没有追问"七十城算不算成" |

但乐毅的三盲有一个特殊之处。

多数人的三盲是"我知道还有很多我不知道的因素"——往外看，越看越觉得自己无知。

乐毅的三盲是**往回看的**：他知道自己已经做了力所能及的一切（善作），但他不把这个等同于"应该成"。他不纠结"我还能做什么"，他接受"我做完了我该做的，成不成不是你我能决定的"。

这不是悲观——这是**在系统条件不足时，不把自己的价值绑定在结果上。**

### 第四层的追问

乐毅全部打下的七十余城，在大燕国换了一个国王之后，几个月内全部丢失。那"善成"了七十城，算成吗？

从结果看，全丢了。从过程看，确实是打下来了。

这触及了"成"本身的定义问题：**成是一个时间点上的状态，还是一个时间段的持续？**

- 如果成是时间点——乐毅成过（七十城一时全属于燕）
- 如果成是时间段——乐毅没成（最后全丢了）

框架的四盲会说：**"成"本身也是临时的——你今天的成可能是明天的败的起点。** 乐毅的七十城打得太顺，燕惠王反而觉得"他这么强，会不会造反"——于是换人，于是全丢。成变成了败的原因。

这不是绕口令——这是校验系统的问题：**你拿到的校验信号（七十城都打了）可能本身就是扭曲的，因为它让你和你的老板都对局势产生了错误的判断。**

## 五、三原则——做事和做成的条件匹配

### 可理解：善作的可理解性很高，善成的可理解性很低

"善作"是可理解的——你有没有能力，看一眼你的战绩就知道。乐毅打了七十城，这个可理解性极高。

"善成"的可理解性很低——你打到了最后两城但没打完，别人怎么看？"差一点就成功了"和"最终还是失败了"是同一个事实的两种描述，取决于你站在什么立场。

**善作的评价尺度是你自己（和你做过的事），善成的评价尺度是别人的预期。** 别人的预期是不受你控制的——燕惠王觉得"怎么还没打完"，他不理解七十城和最后两城的区别。

### 可修理：系统的修理权限不在乐毅手里

乐毅的问题不是他不能修自己的策略——他一直在修（从攻城策略到分化齐国的策略）。问题是他**修不了系统和自己的关系**。

燕昭王在的时候，信任这个条件是好的。燕惠王上台，信任这个条件坏了。乐毅能做什么？

- 他可以去巴结新王——但这不是他的风格
- 他可以主动交出兵权——但那就前功尽弃
- 他可以在信里解释——他确实写了这封信

但所有这些修理尝试都基于一个前提：**新王愿意听。** 如果新王不信任你，你做什么解释都会被当成狡辩。

**可修理的前提是修理的通道是通的。乐毅的修理通道被切断了——不是因为他的方案不行，是因为对方不再接收他的信号。**

### 可自制：乐毅做了唯一可自制的选择

燕惠王派骑劫来换他的时候，他有三个选择：

| 选择 | 结果 | 自制程度 |
|------|------|---------|
| 回燕国接受审判 | 大概率被杀（田单反间计已经生效） | 放弃了自制 |
| 不交兵权、继续打 | 成了造反——从一个功臣变成叛贼 | 另一种被定义 |
| **逃往赵国** | 活下来了，但一生的事业终止了 | **他在两个选项里选了第三个** |

乐毅选了第三条路。不是好选择，是在没有好选择的情况下**自己选择了自己的结局**，而不是让别人替他选。

这是一个在系统条件全面恶化时最关键的框架操作：**当系统不可修、通道被切断、所有选项都不好的时候，你至少还能选"我怎么退出"。**

这和"可自制"的定义完全对应：不是在你被给的选项里选一个最好的，是**当你发现所有选项都不好的时候，你知道你还有不选这些选项的权利。**

## 六、校验系统——善作善成的信号混淆

### 两种校验信号的混淆

"善作"的校验是过程校验："我做对了吗？"
"善成"的校验是结果校验："我得到了我要的结果吗？"

这两种信号经常被混淆：

| | 过程校验（善作） | 结果校验（善成） |
|--|----------------|----------------|
| **问什么** | 我做得对不对？ | 我做成了没有？ |
| **信号来源** | 动作本身的质量 | 系统是否配合 |
| **反馈周期** | 短——每步都能知道 | 长——要等到结果出来 |
| **混淆后果** | 你做对了就觉得应该能成 | 你没成就觉得是做错了 |

乐毅的困境：他每场仗都打得对（过程校验通过），但最终没打完（结果校验没通过）。如果你把结果校验的失败当作过程校验的失败——"你没打赢所以你的打法有问题"——那就错过了真实的教训。

真实的教训是：**过程没问题，但系统条件变了。**

### 吉凶悔吝对应

善作善成和吉凶悔吝的对照：

| 状态 | 对应 | 校验含义 |
|------|------|---------|
| **善作 + 善成** | 吉 | 过程和结果都对——最理想 |
| **善作 + 不善成** | 悔/吝 | 过程对但结果不对——系统条件问题 |
| **不善作 + 善成** | 侥幸 | 过程错但结果碰对了——容易误以为自己行 |
| **不善作 + 不善成** | 凶 | 过程和结果都错——需要全盘反思 |

最有意思的是第三行：**不善作 + 善成**——你没做对但结果好了。这是最危险的情况，因为结果校验给了你一个假的正反馈：你以为自己是对的，下一次继续错。

乐毅的情况是第二行——做对了但没成。这个信号的价值是：**它告诉你做的事情本身没错，但你需要检查系统条件。** 不是改动作，是改条件——或者接受条件不可改，换一个战场。

## 七、照妖镜——谁在用"善作者不必善成"

| 用的人 | 怎么用 | 暴露了什么 |
|--------|--------|-----------|
| **失败者** | "善作者不必善成"——我不是没能力，是条件不好 | 自我安慰——把这句话当成下台阶 |
| **旁观者** | "他也就是个善作不善成的人"——否定过程的质量 | 后见之明——用结果否定过程 |
| **管理者/老板** | "我要的是善成不是善作"——只看结果 | 缩短校验链——把"善成"当成能力问题，忽略系统条件 |
| **乐毅** | 在信里拆开"作"和"成"——我做了我该做的，但系统不让我做完 | **拆锁——不争"我有没有能力"，争"是不是只有能力决定结果"** |
| **你（赌约）** | "砖#3写得很好，但到不了500"——接受善作≠善成 | 正在用这个框架——善作是你对写作质量的控制，善成需要平台推流配合 |

最有意思的是**第三类**（管理者/老板）和**第二类**（旁观者）的合体：

老板说"我只要结果"——听起来很务实，实际上是在缩短自己的校验责任。他把"善成"的压力全部放在做事的人身上，但当条件不配合导致不善成的时候，他只会说"你不行"，而不是"条件不配合，我们一起改条件"。

**这是框架的校验系统论里最核心的一个洞见：如果老板不把自己纳入系统条件的一部分，他永远可以拿"不善成"来否定"善作"——因为失败的条件里永远有一条是"老板没有提供必要的支持"，但评估的时候这条永远被忽略。**

## 八、总结

| 框架概念 | 善作者不必善成对应 | 一句话 |
|---------|----------------|--------|
| **本质主义 → 关系主义** | 能力和结果之间没有必然联系——"不必"是最干净的关系主义操作 | 能力是你自己的，结果是系统和你的关系 |
| **语迷** | "善"字把两件事装进了一个词——善作的"善"是能力，善成的"善"是结果 | 同一个字让你以为它们是同一类事，不是 |
| **盲人摸象** | 三盲的往回看版本——知道自己做了能做的一切，但不把结果绑定在能力上 | 不纠结"我还能做什么"，接受"条件不配合就不是成" |
| **三原则** | 可理解（善作高、善成低）× 可修理（通道被切断）× 可自制（逃往赵国的选择） | 最坏的条件下，你至少还能选怎么退出 |
| **校验系统** | 过程校验（善作）≠ 结果校验（善成）——混淆是最大的坑 | 做对了不等于能成，没成不代表做错了 |
| **照妖镜** | 老板说"我只要结果"——把自己从系统条件里摘出去了 | 失败的原因永远有一条是"老板没支持到位"，但这条从不被写进评估报告 |

> **与[[吉凶悔吝——用认知框架分析]]的对照：** 乐毅的善作而不善成就是吉凶悔吝里的"悔"——做完了回头看发现没成，但你不是因为做错了，是因为系统条件不配合。两篇说的是同一件事：**行动的质量和行动的结果由不同变量控制。** 吉凶悔吝在事中给反馈信号，善作善成在事后做归因分类——一个要及时收手，一个要接受结果。
>
> **与[[鹖冠子——用认知框架分析]]的对照：** 鹖冠子的"天、地、人、命"四稽就是善作的四个维度——你做事之前先在四个维度上校验一遍条件配不配合。乐毅打了七十城但输在"命"（时机变了——新王不信任）。四稽通了三稽，但命不通就不能动。
