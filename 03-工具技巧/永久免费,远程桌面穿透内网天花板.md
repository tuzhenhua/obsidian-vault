
**RustDesk** —— 永久免费、开源、不限时间的远程桌面工具，平替 TeamViewer / 向日葵 / ToDesk。全平台支持（Windows / macOS / Linux / 手机），只要两台设备都能上网，随时随地连回家里或办公室的电脑。

---

## 它凭什么免费？

RustDesk 是开源软件，代码完全公开，没有任何付费墙。它的原理和 TeamViewer 一样，但 TeamViewer 免费版有时间限制、会被识别为商业用途强制断线，RustDesk 没有这些限制。

> 你唯一需要的，就是在两台电脑上各装一个客户端，剩下的它帮你搞定。

---

## 一、安装

### 下载

两台电脑（被控端 + 控制端）都去官网下载对应系统的客户端：

[https://rustdesk.com](https://rustdesk.com)

- **Windows** → 下载 `.exe`
- **macOS** → 下载 `.dmg`
- **Linux** → 下载 `.deb` 或 `.rpm`
- **手机** → App Store / Google Play 搜索 RustDesk

### ⚠️ 安装注意事项

| 注意点 | 说明 |
|--------|------|
| **不要装 Slim 版** | 官网有普通版和 `-slim` 版，被控端务必装普通版。Slim 版缺少服务组件，无法无人值守。 |
| **安装时选 "Install for all users"** | 否则开机自启可能不生效。如果已经装好了，可以事后跑命令补救（见下文配置开机自启）。 |
| **控制端用便携版也行** | 控制端（笔记本/手机）不需要开机自启，装普通版或便携版都可以。 |

## 二、配置被控端（家里/办公室那台电脑）

这是最关键的一步，配置好之后这台电脑就可以无人值守，随时等待你连入。

### 第一步：设置固定密码

打开 RustDesk → 右上角三点菜单 → 设置 → 安全 → 密码 → 改为**"固定密码"**，设置一个你记得住的密码。

> 不设固定密码的话，每次连接都需要有人在那台电脑旁边查看随机密码，失去了远程控制的意义。

### 第二步：开启 OTP 二次验证（强烈建议）

在同一页「安全」设置中，找到**一次性密码（OTP）**选项并开启。绑定你的 authenticator 应用（如 Google Authenticator / Authy / 微软验证器）。

这样一来，即使固定密码泄露，没有手机上的动态验证码也无法连入。

### 第三步：设置开机自启

**Windows：**
- 方法一（推荐）：以管理员身份运行 PowerShell，执行：
  ```
  .\rustdesk --install-service
  ```
- 方法二：任务管理器 → 启动项 → 找到 RustDesk，启用它（但不如方法一可靠，某些系统下可能不生效）

**Linux：**
```bash
sudo systemctl enable rustdesk
sudo systemctl start rustdesk
```

**macOS：**
打开 RustDesk → 设置 → 常规 → 勾选"开机启动"。

> 设置完之后，这台电脑只要开着，不管有没有人在旁边，你都可以从外网连进来。

### 第四步：记下 ID

RustDesk 主界面会显示一串数字 ID，比如 `123 456 789`。把这个 ID 记下来或者拍照，连接时需要用到。

## 三、网络环境要求（文章没提的必备条件）

这是最容易踩坑的地方，先确认你的网络环境：

| 网络类型 | 能否使用 | 说明 |
|----------|----------|------|
| 家用宽带（公网 IP） | ✅ 流畅 | 运营商给的独立公网 IP，直连效果最好 |
| 家用宽带（大内网 / CGNAT） | ⚠️ 能用，但慢 | 大部分手机宽带、部分小区宽带是大内网，依赖官方中继会卡 |
| 公司网络 | ⚠️ 可能被拦截 | 公司防火墙会封锁 RustDesk 流量，需 IT 开白名单，或自建中继改端口 |
| 5G / 4G 热点 | ✅ 能用 | 但延迟偏高，适合轻度操作 |

> **如果连上后卡顿、花屏或频繁断线**，大概率是你的网络是大内网，建议自建中继服务器（见下文）。

## 四、从外网连入

在控制端（你出门带的笔记本或手机）打开 RustDesk，在"远程桌面"输入框里填入刚才记下的 ID，点连接，输入固定密码（如果开了 OTP 还要输入动态码），几秒钟之后对方的屏幕就出现了。

就这样，没有其他步骤。

### 📱 手机端

| 平台 | 可作为控制端 | 可作为被控端 |
|------|:----------:|:----------:|
| Android | ✅ | ✅ |
| iOS | ✅ | ❌ 不支持被控 |

- Android 双向都可以，但屏幕小，精确点击建议配蓝牙鼠标
- iOS 只能**控制别人**，不能**被别人控制**
- 手机操作应急处理文件、跑个脚本够用，长时间办公体验一般

## 五、延迟卡顿怎么办？

连上之后如果画面卡，试试这几个设置：

- **画质** → 控制端设置 → 显示 → 改为"流畅优先"
- **分辨率** → 被控端屏幕分辨率调低到 1280×720，延迟能减少一半
- **音频** → 不需要听声音就关掉，省带宽
- **硬件编码** → 显卡支持的话开启（设置 → 显示 → 硬件编码）
- **帧率限制** → 控制端设置中把帧率降到 10~15fps，操作流畅度反而提升

## 六、安全提醒

| 风险 | 应对方式 |
|------|----------|
| 固定密码暴露 | **开启 OTP 二次验证**，密码泄露也不怕 |
| 数据经第三方服务器 | 默认走 RustDesk 官方中继，数据虽加密但路径在别人服务器上。**隐私敏感必看：自建中继** |
| 被控端无人值守被连 | 不要将固定密码设置为简单组合（如 `123456`），建议 8 位以上字母数字混排 |
| 忘记退出 | 连完后习惯性点断开连接，避免会话一直挂着 |

## 七、更进一步：自建中继服务器

默认情况下，RustDesk 通过它自己的中继服务器帮你转发数据。如果你有一台公网 VPS，可以自建中继，数据只在你自己的服务器和两台电脑之间走，完全在你掌控之下，而且速度飞快。

### 前置要求

- 一台有 **公网 IP** 的 VPS（甲骨文免费绝版了，便宜的可以看 RackNerd / HostDare，一年几十块）
- 防火墙/VPS 安全组放行以下端口：

| 端口 | 协议 | 用途 |
|------|------|------|
| 21115 | TCP | NAT 类型测试 |
| 21116 | TCP + UDP | ID 注册 + 中继服务（核心端口） |
| 21117 | TCP | 直连中继 |

### 简单步骤

1. VPS 上执行安装脚本（两条命令搞定）
2. 客户端设置 → 网络 → ID/中继服务器 → 填入你的 VPS IP

> 自建后延迟通常能从 200ms 降到 20ms 以内，体验接近局域网直连。

## 八、场景建议速查表

| 你的场景 | 推荐方案 | 难度 |
|----------|----------|:----:|
| 家里电脑 ↔ 出门笔记本 | 直接装 RustDesk，用默认中继 | ⭐ |
| 从外面连回家里电脑 | 同上，设置好固定密码 + 开机自启即可 | ⭐ |
| 公司电脑（需远程） | 先确认防火墙不拦截；被拦截则自建中继改端口 | ⭐⭐⭐ |
| 手机操作电脑应急 | 装手机版，配蓝牙鼠标体验更好 | ⭐ |
| 敏感数据 / 隐私要求高 | **必须自建中继**，不要走官方服务器 | ⭐⭐ |
| 远程看视频 / 做设计 | ❌ 任何远程桌面都不适合，帧率和色彩精度不够 | — |

---

> **总结**：普通家用场景照文章操作完全够用。涉及公司网络、隐私敏感或追求流畅体验，需要额外处理网络环境或自建中继。
