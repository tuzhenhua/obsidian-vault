# 探针系统——碎片平台测试方法论

> 从微博打水漂测试中提炼。一套低成本、可复用的方法：怎么在碎片化平台上测试内容有没有人接得住。

---

## 核心思路

不靠分析下结论，用数据说话。把内容拆成两种规格——**瓦片**和**砖头**——分批投放，记录阅读量和互动，对比分析。

- **瓦片**（短金句，<50字）：成本低、数量多、覆盖面广，当探针用
- **砖头**（长论证，300-500字）：深度内容，看平台能不能承载完整论证
- **对照组**：AI自动评论（如微博的罗伯特）——标出哪些反馈是AI给的，哪些是真人的

---

## 投放策略

| 规则 | 原因 |
|------|------|
| 每天一条，不加话题标签 | 避免算法干扰，测裸数据 |
| 不互推、不买量、不叫朋友转 | 排除社交关系链噪音 |
| 观察周期：48小时 | 碎片平台内容生命周期短，48h够看清走势 |
| 瓦片和砖头混合投放 | 同平台对比格式效应 |
| 记录三项指标：阅读量 + 真人评论数 + AI评论 | 区分算法流量和真实互动 |

---

## 判断标准

### 瓦片读数区间

| 区间 | 含义 |
|------|------|
| <5 | 平台几乎不给曝光 |
| 5-15 | 有基础曝光，但无传播力 |
| >15 | 个别内容被算法推了一波 |

### 砖头读数区间（138粉基准）

| 区间 | 含义 |
|------|------|
| <30 | 长内容自然曝光不足 |
| 30-60 | 有稳定读者，格式可行 |
| 60-100 | 内容有自然扩散力 |

### 关键判断

- **砖头 > 瓦片 × 5**：平台奖励完整论证，碎片不是你的主战场
- **砖头 ≈ 瓦片**：说明你的短内容特别强，或者长内容没穿透
- **零真人评论**：不等于失败，只说明没触发互动——可能缺钩子，也可能读者群还没形成
- **AI评论 ≠ 真人评论**：AI只接短抽象内容，不接长文和具体场景。AI接 ≠ 好，AI不接 ≠ 差

---

## 第一轮测试结果（微博，2026.06.02-03）

| 类型 | 数量 | 读数范围 | 均值 | 真人互动 |
|------|------|---------|------|---------|
| 瓦片 | 7 | 5-12 | 8.3 | 0 |
| 砖头 | 2 | 84-91 | 87.5 | 0 |

**结论**：砖头格式有效（纯论证可跑出~90自然阅读），瓦片作探针用但不具独立传播力。平台可测，继续投放。

## 平台机制差异：为什么微博和公众号质量差距这么大

微博和公众号的内容生产端可能是同一批人，质量差距的根源在**分发机制**：

| | 微博 | 公众号 |
|------|------|------|
| 分发逻辑 | 按热度排序，情绪越激烈推得越猛 | 按订阅推送，读者主动选择 |
| 内容形态 | 140字限制，压掉论证空间，只剩立场和金句 | 长文为主，至少要搭起逻辑框架 |
| 读者成本 | 几乎为零，划一下就是一条 | 点开一篇长文要花时间，注意力成本更高 |
| 筛选方向 | 筛出**最不需要思考**的内容 | 倒逼作者**至少把论证搭起来**，否则没人看完 |

所以不是微博上的人更蠢，是微博的**机制筛出了最不需要思考的内容形态**。"10,000倍ASI"这种话在微博上是满分内容——有冲击力、不需要论证、能触发情绪反应。同一句话放公众号，读者会问"这个数字怎么算出来的"。

### 对探针系统的意义

瓦片和砖头的分工恰好对应这两种机制：

- **瓦片**测的是热度机制——短、快、情绪触发强，天然适配微博的算法推流
- **砖头**测的是注意力和论证深度——长、慢、需要读者投入，在公众号上才有读者接得住
- **砖头在微博跑到~90阅读但零评论**，说明平台给了曝光，但读者群体和内容形态不匹配——他们划到了，但不会停下来看完

换平台不只是换场景，是换了一套**筛选器**。以后换新平台，先搞清楚它的分发机制在筛什么方向的内容，再决定投瓦片还是砖头。

---

## 复用指南

换平台、换话题、换形式时，复用这套步骤：

1. **选3-5片瓦片**（该话题下最精炼的短句）→ 测曝光基线
2. **写1-2块砖头**（完整论证，300-500字）→ 测深度内容传播力
3. **投放一周**，每天一条，不加标签
4. **记录数据**：每条48h后的阅读量 + 真人评论
5. **对比分析**：瓦片vs砖头、不同话题、不同发布时间
6. **调整策略**：哪种格式跑得好就加码，跑不动的降级
