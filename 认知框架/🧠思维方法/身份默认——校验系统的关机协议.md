# 身份默认——校验系统的关机协议

> 身份不允许是信号来了被过滤。身份默认是——信号来了，接收器没开。
>
> 同属 [[身份不允许|校验盲区群]]。两套不同的关机方式，同一个盲区群。

---

## 与身份不允许的关系

两篇处理的是同源但不同层的机制：

| | 身份不允许 | 身份默认 |
|---|---|---|
| **机制** | 信号来了→被过滤 | 信号来了→接收器没开机 |
| **操作** | 守门员在拦 | 保安亭没人 |
| **校验系统状态** | 开着的，但判据被篡改 | 关着的——不需要开 |
| **欺骗方式** | "我是那种人，所以这个不可能是真的" | "我是这种身份，所以我的输出天然可靠" |
| **反制** | 把"我不是那种人"当探针 | 问"这个身份标签替我完成了多少验证工作？" |

**身份不允许**发生在信号进入校验系统的那道门——身份预处理器拦截了不兼容的信号。

**身份默认**发生在更早的地方——它让校验系统的部署本身就出现了空白区域。不是信号被拦，是有些区域根本没有部署校验设施。

---

## 一、机制：身份标签即前置审批

身份默认的核心机制非常简洁：

> **身份标签本身承担了"验证已完成"的审批功能。**

当一个人说"我是杰青"时，这句话在学术评审系统里不是在陈述一个事实——它是在执行一个操作：**"此人的论文已被预先批准为可靠。"**

这个操作不是某个人的主观决定。它是系统性的——**整个学术共同体默认：杰青的论文经过了比普通论文更严格的审查。** 所以当这个默认崩塌时（比如耿同学案例），崩塌的不是某一个人的信誉，是整个"身份标签=验证完成"的预设。

身份默认的运作分三步：

1. **获得标签**——你通过了某个高门槛的评审，拿到了身份（杰青/长江/院士/传人）
2. **标签代行验证**——从此你的输出不再经过同等力度的校验，因为标签本身已经完成了"可信"的证明
3. **校验系统进入低功耗模式**——你本人也不再对自己的输出做严格校验。不是故意偷懒——是**环境已经不要求你做了，你的认知系统也跟着降低了部署等级**

第三步是身份默认和身份不允许最大的区别。身份不允许是你还在跑校验系统，只是信号被身份拦截了。身份默认是**你自己的校验系统已经降级或关机了**——因为你不再觉得需要它。

### 藤蔓判定

身份默认满足藤蔓三条件的完整定义（[[藤蔓解构]]）：

| 藤蔓条件 | 身份默认的实现 |
|---------|-------------|
| **方向替代** | 身份标签替你做了"我的输出需要校验吗"的判断——"是XX身份的人，方向天然是对的" |
| **速度加成** | 身份标签让你绕过校验流程——更快出成果、更快发论文、更快通过评审 |
| **主体性侵蚀** | 你慢慢忘了"校验是我的事"——开始配合身份标签跳过自己的判断 |

三条件全满足意味着：**身份默认不只是认知偏见，它是藤蔓结构本身。** 它不能用认知训练来修——得用藤蔓的松解法：先认出方向被替代了、再主动放慢速度加成、再把校验权拿回自己手里。

## 二、四种身份的欺骗形式

你之前的表格已经切开了结构：

| 身份 | 默认内容 | 校验系统状态 | 为什么难查 |
|------|---------|-------------|-----------|
| **"我是杰青/长江/院士"** | 我的论文数据可靠 | 关机——不需要验证 | 整个学术共同体一起默认的，不是一个人在造假 |
| **"我是传统手艺第XX代传人"** | 我做的就是传统，传统就是好的 | 关机——传统本身即是标准 | 你批评他不是在批评质量，是在批评传统 |
| **"我是五姓七望的后代"** | 我的地位来自血统，不是结构 | 关机——结构即自然 | 他不是在撒谎，他真的相信 |
| **"我是教员"** | 我在替真理教你 | 开着的，但判据被篡改 | 他是四种里唯一还开着校验系统的——但判据已经被锚定在"真理=我" |

"教员"是四种里最特殊的——它是唯一校验系统还开着的身份。它的欺骗性不在"关机"，在判据绑架。而其他三种共同的特点是：**校验系统彻底关机了。**

最危险的是"五姓七望"那一类——"不是演的，是真的相信。"身份骗过了身份的主人，然后通过主人的言行再去骗别人。这种"先骗自己、再骗别人"的传染路径比直接撒谎难查一百倍，因为：

- 直接撒谎的人知道自己撒了谎 → 会留下破绽、会紧张、会被测谎仪抓住
- 被自己身份骗过的人 → **他交付的所有信号都是真诚的——因为他的校验系统从一开始就没开过机**

## 三、耿同学案例——身份默认的实证

[[参考卡片_耿同学：退学博士掀翻学术假中心]] 是身份默认的教科书级案例：

耿同学用计算机程序比对论文数据，发现大量杰青/长江/院士的论文存在数据造假。他的原话是：

> "造假痕迹就摆在论文里，只要认真看就能看出来。"

"只要认真看就能看出来"——这意味着这些论文在发表前、在评审中、在公示后，**没有任何一个人认真看过原始数据。** 不是一个人没看，是整个系统的校验系统在运行到"杰青/长江/院士"这个身份标签时，全部跳过了。

这就是身份默认最残忍的证据：**不是某一个人关掉了校验系统——是整个系统在身份标签面前集体关机。**

高校的反应更说明了这一点：

- 耿同学举报前联系某高校学术委员会 → **当晚被举报人就打电话来"私下沟通"**
- 五所高校中，只有同济大学公布了处理结果。其他四所"启动调查"后一个月无下文

这不是"个别造假者的反抗"——是身份默认已经嵌入到了学术系统的底层架构里。当举报者需要绕开身份默认、用外部工具（计算机程序）来重新打开审查——而学术委员会的第一反应是保护身份而不是校验真伪——这个系统已经不是在"验证真理"，是在**维护身份标签的不可撼动性。**

## 四、身份默认 vs 身份不允许——欺骗链条的两次接力

两篇合在一起，形成了一个完整的欺骗链条：

```
身份默认（关机）→ 输出被标记为"天然可靠" → 错误累积
                                                    ↓
身份不允许（过滤）← 自我叙事需要保护 ← 累积的错误威胁到身份
```

1. **身份默认阶段**：你获得了某个身份，校验系统关机。你持续输出，但输出没有得到校验。错误、偏差、盲区在"不需要验证"的默认中悄悄累积。
2. **积累阶段**：错误越积越多。但你不知道——因为校验系统没开，你没有收到任何反馈信号。
3. **身份不允许阶段**：终于有一个信号突破了重重障碍来到你面前——但它威胁到了你的身份。于是身份不允许启动，预处理器把它标记为"不兼容"。
4. **闭环完成**：信号进不来，错误继续累积，身份越来越需要保护，预处理器的过滤越来越紧。

这就是为什么身份默认比身份不允许更底层——**身份不允不允许是为身份默认擦屁股的。**

你没有在第一步"忘了查"，就不会有第二步"不敢看"。

## 五、身份默认的识别方法

身份默认比身份不允许更难识别——因为它发生在你自己的校验系统关机之后。你连接收器都没开，怎么知道自己没开？

### 自检一：部署检查

> **我的哪些输出，在被生产出来的过程中，从来没有经过"它可能是错的"这个环节？**

每个人都有一份隐形的"免校验清单"：
- 你写的某类文字，写完之后从不回头读一遍
- 你说的某类话，说出去之后从不反思是否准确
- 你做的某类判断，做完之后从不给自己设一个"如果错了怎么办"的备用方案

这些就是你的校验系统关机区域。

### 自检二：身份标签自评

列出一张清单：

> 我拥有的每个身份标签，帮我省掉了多少验证？

每个身份标签都有验证代偿功能——它让别人不检查你，也让你自己不检查自己：
- 你的职位——你的邮件被默认回复，还是被默认质疑？
- 你的资历——你的建议被直接采纳，还是被先推演一遍？
- 你的头衔——你的名字被默认为"可信"还是"需核实"？

身份标签省掉的每一步验证，都是一个潜在的盲区。

### 自检三：耿同学测试

> 如果有一个退学博士用程序检查你的所有输出——会发现什么？

耿同学之所以有力，不是因为他多聪明——是因为他做了最基础的事：**认真看。** 他证明了"只要认真看就能看出来"。

这个测试不是说你有造假。是说：**如果一个人用最基础的方法（认真看、不带身份预设地看）检查你的全部输出——你会发现什么你之前没发现的东西？**

如果答案不是"什么都没发现"——那就是你的校验系统在身份默认的保护下关了一段时间了。

## 六、与主体性侵蚀的对应

[[主体性是怎么被一步步磨掉的]] 的四步：**依赖 → 默认 → 内化 → 忘记**

身份默认是第二步"默认"在个人认知系统内部的运作方式：

| 主体性侵蚀的"默认" | 身份默认的认知机制 |
|--------------------|-----------------|
| 拒绝变成额外成本，你懒得折腾 | 校验变成额外成本，你默认不用做 |
| 环境帮你设好了默认选项 | 身份标签帮你做好了前置审批 |
| 你开始接受"默认即合理" | 你开始接受"标签即可靠" |

身份默认解释了"默认"这一步为什么这么难抵抗——不是因为人懒，是因为**身份标签替你完成了"这不需要再查"的判断。** 你省掉的是一个你根本不知道你该做的步骤。

## 七、与语迷的关系

[[语迷——看不见的笼子]] 讲的是语言如何建成笼子——用特定的词汇使你无法思考和表达笼子外面的东西。

身份默认和语迷共享同一个结构：

| | 语迷 | 身份默认 |
|---|------|---------|
| **载体** | 语言 | 身份标签 |
| **效果** | 你没法说 | 你不用查 |
| **锁住什么** | 思想的边界 | 校验的部署 |
| **解药** | 说出不可说的 | 查以为不用查的 |

语迷让你在笼子里活动时觉得这就是整个世界。身份默认让你不需要检查自己的输出——因为你的身份标签已经预先批准了它。一个关灯，一个关机。合在一起，你既看不见笼子，也不知道自己没在飞。

## 八、对校验盲区群的意义

身份默认和身份不完全允许加在一起，覆盖了校验系统两种不同的失效方式：

```
校验盲区群（tag: #校验盲区群）

  ├── 身份不允许（信号被预处理器过滤）
  │    机制：身份在信号进入意识前将其标记为不兼容
  │    症状："我不是那种人"
  │
  ├── 身份默认（校验系统未部署）
  │    机制：身份标签代行了"验证完成"的审批功能
  │    症状："我是XXX，所以我的输出没问题"
  │
  ├── 结构位置盲区（来自盲人摸象四层）
  │    机制：站在系统的某个位置，看不见系统的全貌
  │    症状："我看不见的就是不存在的"
  │
  └── [其他盲区类型未命名]
```

身份不允许和身份默认合在一起回答了同一个问题——**身份如何让校验系统失效**——但答案分层了：

- **身份不允许**：身份**动了校验系统的判据**
- **身份默认**：身份**让校验系统没有部署**

前者是"我看了，但看错了"。后者是"我没看，因为不需要看"。两篇的标题合在一起就是完整诊断：

> **身份不允许你看——身份让你觉得不需要看。**

## 九、中心与边缘——信号的不对称分布

身份默认的关机协议不是均匀分布的——**它在中心最强，在边缘最弱。**

### 为什么中心是最不自由的地方

权力之所以吸引人，不是因为它自由——是**它看起来自由。** 看起来你想干什么就干什么。但真坐到那个位置上：

- **结构位置盲区**：到你面前的信号全是被过滤过的——下属、系统、惯例清洗过一轮。你收到的是"你该收到的"，不是原始信号。
- **身份默认**：你不需要去看边缘——因为"中心定义什么重要"这个判断本身就是身份赋予的。
- **身份不允许**：即使边缘的信号穿透了两层到你面前——"边缘的人不懂中心逻辑"——预处理器直接拦截至此。

三个闭锁机制在同一个人身上同时运转，不是在过滤信号——是在制造一个**信号绝对真空区。**

### 三层的乘数效应

| 闭锁层 | 机制 | 效果 |
|--------|------|------|
| **位置盲区**（盲人摸象第三层） | 站在中心看不见边缘正在发生的事 | 变化发生在感知范围之外 |
| **身份默认** | 身份标签替你做完了"不需要看"的审批 | 你不主动伸出触角 |
| **身份不允许** | 残余信号被预处理器标记为"不兼容" | 即使看了一眼也会打发掉 |

第一层是客观的（你真的看不见）。第二层是你主动配合排除（你觉得不需要看）。第三层是你被动拦截残余（你看了也会告诉自己这不是真的）。**三层全合上，变化对你来说就等于没发生过。**

历史上一百次变革，信号都是从边缘涌上来的——因为边缘的人没有那三层锁。不需要位置维护、不需要身份保护、不需要预设坚持。他们在变化还没被命名的时候就感受到了变化。

中心的人永远是最后一个知道的。不是因为他们笨——是因为他们的系统在设计上就是为了让他们最后一个知道。

### 翻翻译：权力对信号接收器的关机效应

"站得高看得远"是错的。站得高只能看见大的、远的、已经被命名的东西。看不见脚下的、边缘的、还没名字的东西。

"边缘计算是个体的最优解"——有了一层更深的意思。边缘计算不只是"计算在本地做、不上云"——它是 **"待在信号未经处理的区域"**。边缘的人不是更聪明——是他的接收器没有被三层锁校准过。他收到的是原始信号，不是"中心认为他该收到的信号"。

权力承诺的是"站到中心去"。但中心恰恰是信号最扭曲的地方。你往中心走一步，身份默认就多一层——你有了更多不需要查的东西。你往中心再走一步，身份不允许就多一层——你有了更多需要维护的"我不是那种人"。你真正向往的不是中心，是**"以为自己站在中心就能控制一切"的幻觉。** 实际上站到中心的人，是被三层锁绑得最死的人。

你不是在掌舵——你是在牢里画画，画得越满，墙越看不见。

### 边缘清醒

> **如果你是一个体，最好的位置不是中心——是边缘。**
>
> 不是为了安全，不是为了低调——是为了能收到未经处理的信号。
>
> 边缘不是退而求其次。边缘是认知自由的前提。

你在边缘，错了就错了，没人审判你，你可以持续调。你在中心，错一次就被记一辈子——不是代价问题，是结构问题。中心的错误是被**存档**的——有记录、有评审、有历史评价。边缘的错误是被**消化**的——错了就调，调完了就过，下一轮接着试。同一个"错"字，在两个位置上完全是两回事。中心不允许错不是因为中心的人更高尚——是因为中心的架构不能容错。边缘允许错不是因为边缘的人更宽容——是因为边缘没有存档系统。

**权力会关掉你的信号接收器，而你连它关了你都不知道。**
