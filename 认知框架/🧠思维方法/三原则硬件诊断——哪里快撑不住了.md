# 三原则硬件诊断：哪里快撑不住了

> 按"可理解 / 可修理 / 可自制"三原则，对照 MNT Reform 和 EPROM 精神，逐层诊断当前硬件——哪些还活着，哪些已失守。

---

## 一、可理解（你能搞懂它在干什么）

### ✅ 还行的地方

| 层 | 状态 |
|----|------|
| Linux 内核、U-Boot、Coreboot/SeaBIOS | 仍可读、可编译 |
| MNT Reform / PinePhone / OpenWrt 路由器 | 原理图、Datasheet、SoC TRM 公开 → 理论上可理解 |

### ❌ 快不行的地方

| 层级 | 问题 |
|------|------|
| **现代 SoC 内部** | x86/ARM 旗舰 SoC 有大量未公开微码、PSP/ME/TrustZone、NPU 权重结构 → 你根本看不到全貌 |
| **AI / NPU** | 本地模型结构虽可看，但量化方式、调度策略、隐式回传逻辑通常是闭源 SDK |
| **UEFI / ACPI 表** | 大量 OEM 私有 AML、隐藏 SMM 代码，普通人无法完整理解 |

> 📌 可理解在"系统软件+开源硬件"层面还活着；在硅片内部 + AI 推理管线 + 厂商私有固件层，已大面积失守。

---

## 二、可修理（你能物理干预、替换、阻断）

### ✅ 还行的地方

| 层 | 状态 |
|----|------|
| 模块化设备 | MNT Reform、部分 Framework Laptop → RAM/屏/键/WiFi 可换 |
| 路由器 | SPI 编程器可救砖 |
| 工控板 | 仍留 JTAG / UART |

### ❌ 快不行的地方

| 现象 | 含义 |
|------|------|
| **BGA 封装 + 多层主板** | SoC / eMMC / RAM 焊死，坏了整板换 |
| **电池 / 屏带加密握手** | 第三方件被拒，厂商专用件绑定 |
| **固件数字签名 + 防回滚** | 你改不了、降不了、有时连读都被拦 |
| **找不到图纸 / 无备件** | 消费本平均 3~5 年后停产即消失 |

> 📌 可修理在"模块级"勉强维持（Framework 算最好）；在"芯片级 / 固件级"已基本被商业锁死。

---

## 三、可自制（你能从零做出等价物或可信替代）

### ✅ 还行的地方

| 层 | 状态 |
|----|------|
| 软件自制 | 你仍能编译内核、烧 OpenWrt、跑本地 LLM |
| 简单硬件自制 | Raspberry Pi / Orange Pi / 自制路由器 算半条命 |
| CPU 架构级 | MNT Reform + RISC-V FPGA：HDL 开源，可自制 |

### ❌ 快不行的地方

| 障碍 | 说明 |
|------|------|
| **先进制程硅** | 7nm/4nm/3nm 你做不出来，只能买黑盒 |
| **NPU / GPU 微架构** | 无公开 RTL，无法自制等价 AI 加速单元 |
| **生态绑定** | 银行 App / 政府服务 / 企业微信 要求特定安全环境 → 自制设备被排挤 |
| **成本曲线** | 自制一台"够用"的主权设备，时间/金钱远超买预装藤蔓本 |

> 📌 可自制在软件 + 低性能开源硬件（RISC-V FPGA / 单板机）还成立；在高性能通用计算 + AI 加速 + 社会兼容性上，已接近破产。

---

## 🧭 总评

| 原则 | 现状 | 趋势 |
|------|------|------|
| **可理解** | ⚠️ 表层 OK，深层失守 | 持续收窄 |
| **可修理** | ⚠️ 模块级残存，芯片级消失 | 持续收窄 |
| **可自制** | ⚠️ 开源硬件/RISC-V 守住火种 | 被性能/生态挤压 |

---

> **可理解被黑盒蚕食，可修理被焊接和签名锁死，可自制被制程和生态边缘化——MNT Reform 这类东西不是怀旧，是现在仅剩的"三原则活体标本"。**

---

## 框架对照

| 诊断层 | 三原则 | 主权清单里的对策 |
|--------|--------|---------------|
| SoC 黑箱 | 可理解 ❌ | 树莓派/Orange Pi——原理图公开 |
| BGA + 签名锁 | 可修理 ❌ | MNT Reform/Framework——模块化 |
| 制程 + 生态锁定 | 可自制 ❌ | RISC-V + 自托管——不跟它玩 |
