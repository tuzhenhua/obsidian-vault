# 投资研究系统

> 不是预测市场。是**在不确定中做有纪律的决策。**

---

## 一、核心理念

你能控制的 vs 你不能控制的：

| ✅ 你能控制 | ❌ 你不能控制 |
|-----------|-------------|
| 买什么 | 它什么时候涨 |
| 多少钱买 | 政策什么时候出 |
| 仓位多重 | 游资什么时候砸 |
| 什么时候走（你的止损线） | 别人什么时候跑 |

**这个系统只负责左边那一列。**

## 三、每篇个股笔记的结构

每篇笔记统一用这个框架——**决策在前，跟踪在后，复盘收尾。**

### 1. 基础信息（YAML 前置元数据）

```yaml
---
tags: [投资/个股]
stock_name: 股票名称
stock_code: 代码
industry: 行业
buy_date: 
buy_price: 
position: 仓位占比
status: 持有/已清仓/观察
---
```

### 2. 买入逻辑（最重要的部分）

这是整个系统的核心。**买的时候想的什么，老老实实写下来。**

- **投资主题**——一句话说清楚买它是因为什么（产业趋势？困境反转？估值修复？套利？）
- **商业模式**——它靠什么赚钱？这钱好不好赚？
- **护城河**——为什么别人抢不走它的生意？
- **估值**——买入时的PE/PB/股息率，跟历史比、跟同行比贵不贵？
- **催化剂**——预计什么事件会让市场重新定价它？
- **风险点**——如果错了，最可能是哪里出了问题？

### 3. 跟踪日志

流水账，越短越好，关键是记录**逻辑是否还在**：

| 日期 | 事件 | 逻辑验证/证伪 | 操作 |
|------|------|-------------|------|
|  |  |  |  |

### 4. 卖出复盘

卖出不是结束——**复盘才是。**

- **卖出日期/价格**
- **盈亏**
- **卖出的原因**（逻辑变了？达到目标价？止损？需要钱？）
- **复盘结论**：
  - 买入逻辑是对的吗？如果错了，错在哪里？
  - 执行层面有没有问题？（止损没执行？仓位过重？）
  - 下次怎么做不一样？

### 5. 心理笔记（可选，但强烈推荐）

记录买入、持有、卖出时的情绪状态。这一部分不对外，只对自己诚实。

## 五、纪律守则（自定）

这些是你自己定给自己的规则。写下来就是承诺。

- 单只股票仓位不超过 __%
- 单只股票止损线 __%
- 买入前必须写完买入逻辑，否则不买
- 每月/每季至少 review 一次持仓逻辑
- 卖出后必须写复盘，否则不开新仓

## 六、常见失败模式（持续更新）

看到自己的错误重复出现，就记在这里：

1. **追涨**——涨了觉得"还能涨"，忘了买入逻辑
2. **不想卖**——跌了补仓、再跌再补，把短线做成了股东
3. **信息过载**——看太多分析，忘了自己当初为什么买
4. **完美主义**——想等最低点买、最高点卖，结果什么都没做
