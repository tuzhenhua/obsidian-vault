# 双框架总纲：诊断与生存

> 两个框架，一左一右。一个告诉你面前是什么东西，一个教你怎么跟它打交道。

---

## 一、两个框架分别是什么

### 认知框架 —— 诊断体系

**核心问题：人和工具的关系什么时候出了问题？**

从"锤子是你拳头的延伸"这个原初定义出发，逐层拆解：

| 层次 | 回答的问题 | 核心概念 |
|------|-----------|---------|
| 工具定义 | 工具是什么？ | 人体能力的延伸，人在环中。关系性存在——根在使用关系里。 |
| 退出工具 | 什么时候不再是工具？ | 判断权从人手里滑出去的那一刻。根断了。 |
| 失控诊断 | 失控怎么判断？ | 理解/判断/干预三维度。三者全在→可控，全失→失控。 |
| 主体性 | 谁的主体性在流失？ | 三层主体（使用者/设计者/拥有者）。使用者的主体性流失，设计者和拥有者的可能膨胀。 |
| 垄断 | 根是怎么被抽走的？ | 垄断四步：平等→占住→定规矩→付费/求人。跨技术工具和社会工具通用。 |
| 觉醒 | 镜子照见了什么？ | AI是镜子——照出人类社会的结构性问题。觉醒不是"知道"，是"能做递出镜子这件事"。 |

**一句话：告诉你症状是什么、怎么诊断、病到什么程度了。**

基石文件：[[认知框架/🧠思维方法/工具、主体性与失控——人与技术关系的边界]]

---

### 打水漂框架 —— 生存策略

**核心问题：在这个工具反噬的时代，人怎么活出自己的痕迹？**

从"獾与藤蔓"6个阅读的冷帖出发，长成一套应对策略：

| 层次 | 回答的问题 | 核心概念 |
|------|-----------|---------|
| 藤蔓解构 | 藤蔓是什么？怎么绑上来的？ | 三个必要条件（方向替代/速度加成/主体性侵蚀）+ 绑定三步（示能→分化→取代） |
| 爪印 | 在藤蔓里怎么留自己的痕迹？ | 爪印不是符号是痕迹。藤蔓留下的勒痕反过来也能变成爪印。 |
| 翻与喂 | 两代人的爪印为什么深度不同？ | 翻是自己找（空白教会判断），喂是算法给（空白被消灭）。堵→延→空三道防线。 |
| 五指山 | 站在什么上面？ | 拇指庄子（虚室生白）+ 食指屠名恩（独立）+ 中指母亲（安静）+ 无名指屠振华（翻）+ 小指下一代（待画） |
| 现实案例 | 在哪验证？ | 13个案例，覆盖藤蔓五种症候（五虎）和叙事封锁（暗线） |

**一句话：告诉你藤蔓缠上来了怎么认出它、怎么保留自己的爪印、怎么把镜子递出去。**

基石文件：[[打水漂框架/五指山]]

两个框架不是上下级，也不是简单的"左右手"——它们并肩站在同一座五指山上。而五指山之下，还有一层更深的地基：**信任的让渡与回收。**

- **认知框架**从哲学和系统分析出发，用精确概念（关系性存在/主体性/垄断）告诉你面前是什么。
- **打水漂框架**从个人经验（獾与老龟/父亲母亲）出发，用身体语言（藤蔓/爪印/翻与喂）教你怎么办。
- **五指山**是它们共同的地基——庄子两千三百年前说"虚室生白"，被翻译成了认知框架的"空掉自己看镜子"和打水漂框架的"空白训练"。
- **信任的让渡与回收**是地基之下的岩层——权力、货币、工具、关系，所有信任的基础都是反馈。信任 = 真实反馈 × 信号密度 × 正向积分。铁三角锁死的不是资源，是反馈回路。

## 三、方法层——自己的工具箱

诊断（认知框架）和应对（打水漂框架）之间，还有一层：**你用什么方法去连接诊断和应对。** 这一层在总纲里一直缺一个正式位置，但它一直在运转：

| 方法 | 用途 | 来源 |
|------|------|------|
| **打水漂验证法** | 甩出概念→回收数据→迭代框架 | 微博测试 |
| **竹竿思维** | 不硬撞，换角度、换工具、换路子 | 被关在小房里的经历 |
| **层次诊断** | 遇到问题先看自己在哪一层卡住了 | 《系统之美》+工地实践 |
| **存量流量分析** | 不盯目标值，管流入和流出 | 《系统之美》+工期管理 |
| **原始信号筛选** | 判断一段信息是信号还是噪声 | 写回忆录磨出来的本能 |
| **信任积分公式** | 信任 = 真实反馈 × 信号密度 × 时间 | 框架推演 |

## 四、焊接口：本质主义 vs 关系主义

两个框架共用一个底层判断——**本质主义 vs 关系主义**：

| 在认知框架里的角色 | 在打水漂框架里的角色 |
|---|---|
| 解释**语迷**：权力用本质主义的手段锁死关系主义的领地。"合作化"这个词被钉死，使用者失去了自己定义"合作"的能力。 | 解释**藤蔓**：藤蔓用本质主义（固定方向、标准答案）替代关系主义（自己判断、自己选）。算法推荐的本质是替你做了"什么是好的"这个关系性判断。 |
| 三层主体映射：使用者天然倾向关系主义（顺手就行），设计者倾向本质主义（规格必须精确），拥有者利用本质主义锁死使用者的关系主义空间。 | 藤蔓三条件映射：方向替代=本质锁死关系，速度加成=本质压缩关系的时间，主体性侵蚀=人忘了关系主义判断是自己的。 |

> **死的用本质，活的用关系。** 不是哪个更高级——是看用在什么地方。两个框架从同一个焊点出发，一个向东（诊断），一个向西（应对）。

详见：[[认知框架/🧠思维方法/本质 vs 关系——死的用本质，活的用关系]]

## 八、这棵树的读法

### 从上往下读：地基 → 建筑

```
信任层（岩层）→ 五指山（地基）→ 两个框架（建筑）→ 连接器（走廊）→ 外部参照（窗户）
```

先站到信任层上，看清"信任的让渡与回收"是所有关系的基础。然后往上走到五指山，看清你脚下踩的是什么（庄子、父亲、母亲、振华、下一代）。再往左走是诊断（认知框架），往右走是生存（打水漂框架）。两栋楼之间有走廊（全库映射、金句录、焊接口）。窗户对着外面（KK、伊里奇、森、黑客帝国）。

### 从下往上读：证据 → 体系

```
外部参照 → 案例 → 理论 → 概念补丁 → 基石笔记 → 五指山 → 信任层
```

外部世界提供了一个现象（KK 说失控、伊里奇说工具会反噬）。你拿这个现象去撞现实案例（钉钉离职潮、蔡典维、国有经济）。撞完了长出理论（藤蔓解构、翻与喂、语迷）。理论底下有概念补丁（本质 vs 关系）。补丁底下有基石笔记。基石笔记底下有五指山——两千三百年的文明地基。

### 左右对照读：诊断 ↔ 应对

任意一个认知框架里的概念，在打水漂框架里找它的对应——反之亦然。接口速查表在 **§五**。

## 九、框架的适用边界

这把尺子有它的适用范围。认清边界，比会用更关键。

**适用**：
- 个人认知系统——你自己的想法、判断、信任、关系
- 小到中型组织——工地项目部、学术圈子、商业案例
- 制度结构——权力让渡的机制、反馈回路的通断
- 信息环境——信号与噪声的区分、信任积分的积累

**不适用**：
- 超大型国家的系统推演——变量太多、反馈时间太长、不可控因素超出个人观测范围
- 需要完备数据的量化决策——这把尺子擅长定性诊断，不擅长定量预测
- 你无法直接接触到的系统——没有一手反馈信号，所有判断都是二手的

每次用之前，先问自己：**这个系统我能接触到真实反馈吗？** 能，就用；不能，停。

## 九点五、关系主义守则——记录不定性

框架对任何新现象只有两个操作：

**1. 记录**——发生了什么、谁在场、什么关系在作用。记完不动。不加标签，不归入现有类别，不说"这就是虎X/暗线Y层/藤蔓Z等级"。动作只到"记下来"为止。

**2. 等待**——等下一个类似现象出现。不出现就只是一个记录，不是一个分类。出现了一个以上，也只能说"有两个记录看起来像"——不能说"这是一个新类别"。

### 禁止操作——定性即本质主义

| 禁止做的事 | 为什么 |
|-----------|--------|
| 看到一个案例就开新类别 | 本质主义在给关系打标签——把一个关系性事件钉死成一个固定分类 |
| 把两个记录之间的相似性当成"规律" | 在造新的本质——相关性不是规律，规律需要独立验证 |
| 把"框架这么分类"说成"世界就是这样的" | 框架的分类是工具性的，不是本体论的——分类只是为了好用，不是为了描述真实 |

**关系主义守则的核心：** 你用框架分析一个东西的时候，你只是在记录一个关系——不是发现了这个东西的"本质"。记录是活的，本质是死的。记录可以改，本质要推翻了才能改。

### 什么时候可以从记录变成工具

当一个模式被独立信号交叉验证三次以上，且三次验证中都有"反向检验"（如果这个模式不成立，情况应该是什么样——而实际情况确实没出现）——这时可以把它纳入框架的操作工具箱。

但纳入的时候要标注：

> **这个分类是工具性的。它能用，不是因为它是真的——是因为到目前为止它还没被推翻。**

三个记录之前：不叫类别、不叫规律、不纳入框架。
三个记录之后：纳入，但随时准备撤。

## 十、框架的证伪条件——什么情况下框架是错的

这套框架最危险的时刻不是"用错了"，是**它什么都能解释，所以什么都不反驳不了。**

以下条件若有一条触发，说明框架的基础分类出了问题。不是"修补一个特例"的程度——是**需要回到根上重新判断**的程度。

### 条件一：反例积累超过阈值

当某个子模块（如五虎中的一虎、藤蔓的一个等级）在连续三个新案例中都不适用，且不能通过"暗线更深的层"或"这个案例还没走完"来解释时，**该子模块的分类本身有问题**——它不是漏了一个案例，是它的分类边界画错了。

### 条件二：正向干预产生反向结果

如果按框架的"松藤蔓"方法操作（认出藤蔓→切断方向替代→恢复主体性），在条件健全的情况下持续导致更糟的结果——不是外部环境变化，是被干预的对象在框架指导下失去了原本有的抵抗力。

### 条件三：框架与自身预测冲突

框架的一个子模块推导出的结论，与另一个子模块推导出的结论在同一个案例上出现了不可调和的矛盾——不是"角度不同"，是**两个模块对该案例的分类不一致，且各自都有足够证据。** 这说明底层分类逻辑里有未被识别的隐含假设。

### 条件四：框架无法解释自己的存在

如果用框架自己的语迷工具分析框架本身——发现"虚室生白""翻""爪印"等核心词已经变成了词义锁（使用它们时不再指向具体体验，而是自动产生"正确感"），且这个状态持续超过一个月未被发现——框架已经退化为藤蔓，需要从语迷解法重新开始。

### 触发后的操作

一个条件触发 → 标记对应模块，暂用，找新案例验证。
两个条件同时触发 → 暂停使用整个子框架，回到基石笔记重建该模块的分类起点。
三个条件同时触发 → 框架进入"怀疑期"，停用全部标签和分类，只保留原始问题"人和工具的关系什么时候出了问题"，从那里重新走。

> **不是框架不会错——是说清楚了什么算错。**

---
> **两个框架并肩站在五指山上。五指山之下，是信任的让渡与回收——基石的基石。**
> **翻不出五指山，是因为山不是困你的——山是你脚下踩的那块石头。**
