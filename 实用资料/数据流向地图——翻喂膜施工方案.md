## 三、每节衬管的验收标准

每接完一节，过这三条：

| 验收 | 问自己 |
|:--|:--|
| **数据在哪** | 能在地图上指出来——"这个存在 D 盘 / 这个在 GitHub / 这个在 Bitwarden 云端" |
| **谁能读** | 只有我 / GitHub 公开 / DeepSeek 服务器 / 不知道 |
| **喂不喂回去** | 这个工具把我的翻当它的喂吗？如果不知道——默认是 |

---

## 四、不做的

| 不做什么 | 为什么 |
|:--|:--|
| 不用一次性全换掉 | **一节一节接。** 和工地上做进度对照一样——做完一段验收一段。不急 |
| 不追求完美隔离 | 不可能完全断网。目标不是"建一个独立王国"——是**翻和喂不共用同一根管道** |
| 不自己造轮子 | 用现成的开源工具。本地模型用 ollama/LM Studio，自建 Git 用 Gitea。**能用现成的就用现成的** |
