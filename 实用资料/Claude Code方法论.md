# 刘小排 Claude Code 方法论

来源：ClawX全球生态伙伴（2026.06.08）

刘小排——全球Claude Code用量最大的开发者，被官方点名认可。

## 核心认知

把Claude Code当**一个需要管理的初级工程师**，不是搜索引擎也不是魔法棒。管好是神兵，管不好是灾难。

## 六条方法论

### 1. CLAUDE.md 是命根子
项目根目录放CLAUDE.md，写入职手册——技术栈、目录结构、编码规范、禁止事项。一次写好，每次对话白捡效率。

### 2. 先规划再动手
Plan Mode（Shift+Tab）只出计划不碰文件。确认方向后再执行。多花30秒看计划，少花30分钟擦屁股。

### 3. 任务切小
一次只干一件事。大任务先拆成小步骤，一步步喂。任务越具体，跑偏概率越低。

### 4. 控制对话长度
上下文用到50%左右果断开新会话。重要约定写进CLAUDE.md，别靠对话硬记。

### 5. Git 频繁提交
每完成一个小功能立刻commit。配合 git worktree 多终端并行。

### 6. 会问问题比会改代码重要
- 给角色定位："你是一个有10年经验的Go后端工程师"
- 加边界约束："不要修改测试文件"
- 要求先列计划再动手
- 指定验证步骤

## 一句话总结

**CLAUDE.md管记忆 → Plan Mode管方向 → 小任务管节奏 → Git管安全 → worktree管并行**

> 这不是玄学，是系统。不是运气，是流程。
