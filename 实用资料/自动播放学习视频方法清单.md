## 自动播放学习视频方法清单

### 前置判断：先确认平台检测机制
不同平台检测强度不同，决定用什么方案：

| 平台类型 | 检测手段 | 推荐方案 |
|---------|---------|---------|
| 通用视频站 | 基本只查进度条 | 油猴脚本 |
| 学习强国 | 弹窗确认、时长统计、IP记录 | 专用脚本 + 模拟真人 |
| 法宣/干部网院 | 防拖动、期末确认、部分有人脸 | Selenium自动化 |
| 企业内部系统 | 看IT水平，有的只查完成率 | 开发者工具脚本 |

### 四种方案（按难度排序）

**① 浏览器开发者工具 Console 脚本**
```
setInterval(() => { document.querySelector('.play-button')?.click() }, 1000)
```
- 优点：零安装，浏览器自带
- 缺点：选择器需根据平台调整，遇到弹窗要改代码

**② 浏览器插件**
- Video Speed Controller（倍速播放，最高16x）
- Auto Skip / Auto Confirm 类插件（自动确认弹窗）
- 优点：装好即用，无需写代码
- 缺点：部分平台的弹窗检测机制会绕过

**③ 油猴脚本（Tampermonkey）**
- 去 greasyfork.org 搜平台名称，找现成脚本
- 优点：社区维护，适配好
- 缺点：需装Tampermonkey扩展

**④ Python Selenium 自动化**
- 写脚本模拟完整操作：打开→播放→等进度→翻页
- 优点：最全面，可模拟真人操作习惯
- 缺点：需要写代码，门槛最高

### 注意事项
- 后台有日志：IP、观看时长、弹窗响应间隔都可能被记录
- 部分有视频中间段人脸识别或验证码，脚本容易被发现
- 先摸清平台的检测严格程度再选方案
